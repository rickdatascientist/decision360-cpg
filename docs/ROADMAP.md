# Evidence-gated roadmap

This is a sequence of gates, not a promise of features or dates.

1. Validate buyer, decision, current failure, and five workflows.
2. Specify a deterministic evidence → risk → recommendation → approval → outcome contract.
3. Build one end-to-end vertical slice before adding an agent layer.
4. Add allow-listed Gemini/ADK tools, structured outputs, refusal rules, cost, and trace capture.
5. Run a 60-case evaluation suite and three unguided product tests before public beta.

Every gate requires a linked artifact, a named reviewer type, a recorded decision, and a stated failure response.
